# 图形界面使用指南

## 启动方式

### 方法 1：双击启动（推荐）

直接双击 `launch_gui.bat` 文件

### 方法 2：命令行启动

```bash
# 使用启动器
python launch_gui.py

# 或直接运行（使用 ArcGIS Pro Python 效果更好）
"C:\Program Files\ArcGIS\Pro\bin\Python\envs\arcgispro-py3\python.exe" benchmark_gui.py
```

```
┌─────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────┤

├─────────────────────────────────────────────────────────────┤
│                     步骤区域                                 │
├─────────────────────────────────────────────────────────────┤
│                     日志区域                                 │
├─────────────────────────────────────────────────────────────┤
│                     状态栏                                   │
└─────────────────────────────────────────────────────────────┘
```

### 测试设置区域

在"测试设置"区域可以配置：

#### 循环次数
- **默认**: 3 次
- **范围**: 1-20 次
- **说明**: 每项测试重复执行的次数，用于计算平均值和标准差
- **建议**: 
  - 快速测试: 3 次
  - 学术论文: 5-10 次

#### 预热次数
- **默认**: 1 次
- **范围**: 0-5 次
- **说明**: 正式测试前不计时的预热运行，用于消除冷启动影响

#### 数据规模
- **选项**: 中小型 / 大型
- **默认**: 中小型
- **说明**: 
  - 中小型: 适合脚本验证（约需 5GB 磁盘空间）
  - 大型: 适合学术论文（约需 50GB 磁盘空间）

### 步骤区域

每个步骤包含：
- **步骤编号**：如"步骤 1"
- **步骤名称**：如"验证环境"
- **步骤描述**：该步骤的具体说明
- **状态显示**：等待执行/执行中.../成功/失败
- **执行按钮**：点击开始执行该步骤

### 快捷操作按钮

- **执行所有步骤**：按 1→2→3→4→5 顺序自动执行（遇到失败会停止）
- **重置所有状态**：将所有步骤状态恢复为"等待执行"
- **导出报告**：将生成的报告导出到指定目录

### 日志区域

显示实时执行日志，包括：
- 时间戳
- 日志级别（INFO/WARN/ERROR）
- 详细信息

日志操作按钮：
- **清空日志**：清空当前日志显示
- **保存日志**：将日志保存到文本文件

### 状态栏

显示当前状态：
- 就绪
- 正在执行...
- 执行完成

右侧有进度条动画，表示正在执行中。

## 使用流程

### 标准流程

```
步骤 1: 验证环境 ──► 步骤 2: 生成数据 ──► 步骤 3: Py2.7测试
                                                    │
步骤 5: 分析结果 ◄── 步骤 4: Py3.x测试 ◄─────────────┘
```

### 详细步骤

#### 步骤 1：验证环境

点击"步骤 1"右侧的【执行】按钮

**功能**：
- 检测 Python 2.7 是否安装
- 检测 Python 3.x 是否安装
- 检查 arcpy 模块是否可用

**成功标志**：状态变为"成功"，弹出成功对话框

**失败处理**：
- 检查 ArcGIS 是否正确安装
- 检查 Python 路径是否正确
- 查看日志了解详细错误

#### 步骤 2：生成测试数据

点击"步骤 2"右侧的【执行】按钮

**功能**：
- 创建渔网多边形（1000×1000 网格）
- 生成随机点（10万点）
- 创建测试用栅格数据

**注意**：
- 只需执行一次
- 需要足够的磁盘空间（约 5GB）
- 耗时约 5-15 分钟

#### 步骤 3：运行 Python 2.7 测试

点击"步骤 3"右侧的【执行】按钮

**功能**：
- 使用 ArcGIS Desktop 的 Python 2.7 运行基准测试
- 执行 12 项测试（6项矢量 + 4项栅格 + 2项混合）
- 每项测试运行 5 次，记录平均值

**耗时**：约 30-60 分钟（取决于硬件性能）

**注意**：
- 请确保 ArcGIS Desktop 许可可用
- 测试期间请勿关闭 ArcGIS 相关进程

#### 步骤 4：运行 Python 3.x 测试

点击"步骤 4"右侧的【执行】按钮

**功能**：
- 使用 ArcGIS Pro 的 Python 3.x 运行基准测试
- 执行同样的 12 项测试

**耗时**：约 30-60 分钟（取决于硬件性能）

**注意**：
- 请确保 ArcGIS Pro 许可可用
- 测试期间请勿关闭 ArcGIS Pro

#### 步骤 5：分析结果并导出

点击"步骤 5"右侧的【执行】按钮

**功能**：
- 对比 Python 2.7 和 Python 3.x 的测试结果
- 生成统计报告（加速比、标准差等）
- 生成多种格式的报告文件

**生成的文件**：
- `comparison_report.md` - Markdown 格式报告
- `comparison_table.tex` - LaTeX 表格
- `comparison_data.csv` - CSV 数据文件
- `comparison_data.json` - JSON 原始数据

#### 导出报告

点击【导出报告】按钮

**功能**：
- 选择保存目录
- 将报告文件复制到指定位置
- 方便用于论文写作或分享

## 一键执行

点击【执行所有步骤】按钮，将自动按顺序执行：

```
步骤 1 → 步骤 2 → 步骤 3 → 步骤 4 → 步骤 5
```

**特点**：
- 自动顺序执行
- 如果某一步失败，会停止后续步骤
- 可以在任意步骤后暂停

**适用场景**：
- 首次运行测试
- 完整测试流程
- 无人值守执行

**注意**：
- 总耗时约 1.5-3 小时
- 请确保电脑在此期间保持开机
- 建议连接电源（笔记本）

## 常见问题

### Q: 界面显示乱码怎么办？

**A**: 确保系统支持 UTF-8 编码，或使用英文版 Windows。

### Q: 点击执行后没有反应？

**A**: 
1. 检查是否有杀毒软件阻止
2. 检查 Python 路径是否正确
3. 查看日志了解详细错误

### Q: 步骤 3 或步骤 4 执行很慢？

**A**: 这是正常的，因为：
- 需要执行 12 项测试
- 每项测试运行 5 次
- 涉及大量数据处理

请耐心等待，大型数据处理可能需要 30-60 分钟。

### Q: 如何调整测试数据规模？

**A**: 在 GUI 的"测试设置"区域，可以直接修改：
- **循环次数**: 1-20 次（默认 3 次）
- **预热次数**: 0-5 次（默认 1 次）
- **数据规模**: 中小型 / 大型

或者手动编辑 `config/settings.py` 文件。

### Q: 导出报告失败？

**A**: 
- 检查是否有写入权限
- 检查磁盘空间是否充足
- 确保已完成步骤 5

### Q: 可以同时运行多个步骤吗？

**A**: 不建议。步骤之间有依赖关系，请按顺序执行。

## 最佳实践

1. **首次使用**：按顺序执行步骤 1→2→3→4→5
2. **重复测试**：步骤 2 只需执行一次，可直接从步骤 3 开始
3. **修改数据规模**：修改配置后，需要重新执行步骤 2
4. **保存日志**：建议每次测试后保存日志，便于排查问题
5. **导出备份**：及时导出报告，防止数据丢失

## 故障排除

### 问题：GUI 无法启动

**解决步骤**：
1. 检查是否安装了 ArcGIS Pro 或 ArcGIS Desktop
2. 尝试命令行方式：`python launch_gui.py`
3. 检查 Python 路径是否正确
4. 查看是否有错误提示窗口

### 问题：步骤 1 失败（环境验证）

**解决步骤**：
1. 确认 ArcGIS 已正确安装
2. 检查 Python 路径：
   - Python 2.7: `C:\Python27\ArcGIS10.8\python.exe`
   - Python 3.x: `C:\Program Files\ArcGIS\Pro\bin\Python\envs\arcgispro-py3\python.exe`
3. 检查 ArcGIS 许可是否有效

### 问题：步骤 2 失败（生成数据）

**解决步骤**：
1. 检查磁盘空间（至少 5GB 可用）
2. 检查数据目录写入权限
3. 查看日志了解详细错误

### 问题：步骤 3 或 4 失败（运行测试）

**解决步骤**：
1. 检查 ArcGIS 许可级别（需要 Advanced）
2. 检查内存是否充足（建议 16GB+）
3. 检查测试数据是否已生成
4. 查看日志了解详细错误

### 问题：步骤 5 失败（分析结果）

**解决步骤**：
1. 确保步骤 3 和步骤 4 已成功执行
2. 检查结果文件是否存在：
   - `results/raw/benchmark_results_py2.json`
   - `results/raw/benchmark_results_py3.json`
3. 查看日志了解详细错误
