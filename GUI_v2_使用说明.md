# GUI 使用说明（面向 v1.0.0）

> 文件名沿用历史命名，内容以当前发布版 GUI 为准。

## GUI 能做什么

- 统一运行 Python 2.7、Python 3.x 与开源库对比测试。
- 勾选多个数据规模，一次批量出多套结果。
- 显示总进度、当前任务与 ETA。
- 自动检测开源库是否可用，并提供安装/重检按钮。
- 保存最近一次配置，下次启动自动恢复。
- 支持中英文切换与界面字体缩放。

## 启动方式

### 推荐

双击：

```text
启动工具.vbs
```

### 兼容方式

```bash
python launch_gui.py
python benchmark_gui_modern.py
```

### 备用方式

- `launch_gui.bat`
- `launch.vbs`
- `start_gui_hidden.vbs`
- `启动工具_系统Python.vbs`

## 主界面说明

### 顶部区域

- 显示工具标题与副标题。
- 右侧语言按钮可在中文/英文之间切换。

### 左侧设置区

- Python 2.7 路径与 Python 3.x 路径。
- 测试运行次数与预热次数。
- 多规模勾选区域：`tiny / small / standard / medium / large`。
- 当前编辑规模的参数面板，可分别调整矢量与栅格参数。
- 临时数据目录与是否使用时间戳文件夹。

### 选项开关

- 多进程测试。
- 开源库测试。
- 开源库状态显示、重检与安装按钮。

### 快捷操作

- 开始测试。
- 停止测试。
- 打开生成结果文件夹。
- 日志工具栏：`Clear / Save / Copy`。

### 进度与日志

- 总进度条展示整体完成度。
- 当前任务区域显示正在运行的测试。
- ETA 会根据已完成任务动态估算。
- 日志区会输出环境验证、运行命令、结果文件位置与异常信息。

## 高级设置对话框

通过“高级设置”按钮可以进入设置对话框，主要包含以下内容：

- Python 路径浏览与环境验证。
- 测试运行次数、预热次数、多进程 worker 数。
- 五级数据规模参数编辑与恢复默认值。
- 保存结果选项、时间戳文件夹、保留天数。
- 是否记住上次设置与界面字体缩放。

配置默认保存在：

```text
%USERPROFILE%\.arcgis_benchmark\config.json
```

## 推荐使用流程

### 首次使用

1. 启动 GUI。
2. 检查 Python 2.7 与 Python 3.x 路径。
3. 如需开源库对比，先确认状态为“已安装，可启用”。
4. 选择 `tiny` 规模做一次快速验证。
5. 确认报告文件生成正常后，再跑 `medium` 或 `large`。

### 日常使用

1. 勾选需要的规模。
2. 按需启用“多进程”或“开源库”。
3. 点击“开始测试”。
4. 完成后点击“打开生成结果文件夹”查看输出。

## 输出目录

默认根目录：

```text
C:\temp\arcgis_benchmark_data\<时间戳>\
```

每个规模单独生成：

```text
<规模>\
├── comparison_report.md
├── comparison_table.tex
├── comparison_data.csv
├── comparison_data.json
└── data\
    ├── py2\
    ├── py3\
    └── os\
```

## 常见问题

### 启动后没有界面

- 优先使用 `启动工具.vbs`。
- 如果仍无响应，改用 `python launch_gui.py` 查看是否有报错。

### 开源库复选框不可勾选

- 说明 Python 3.x 环境中缺少 `geopandas / rasterio / shapely / pyogrio / numpy`。
- 可点击“安装开源库”自动处理。

### 想中途停止

- 点击“停止测试”即可请求停止。
- 已完成的结果文件会保留，未完成部分不会继续执行。

### 结果文件在哪

- 直接点击“打开生成结果文件夹”。
- 或到 `C:\temp\arcgis_benchmark_data` 下按最新时间戳查找。
